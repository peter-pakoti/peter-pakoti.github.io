What are the best practices associated with using classes vs. IDs?

Well lets firstly start with explaining what classes and IDs are!

Both IDs and classes are used in a way to target a specific HTML element in our pages in order to give them a specific styling.

When you're working with CSS, there are no specific reasons forcing you to use an ID over a class. However, it is best practice to only use IDs if you want a style to apply to one element on the web page, and to use classes if you want a style to apply to multiple elements.

To simplify this, it really is up to you! However if you want to improve your efficiency and keeping your code organised and tidy then I suggest you use these to help do so.

Now lets take a look at some examples to give you a better understanding of what I'm talking about!

In 'Image-A' you can see that we have some text in our p-tag. If we were to add changes to the p-tags style using our css file it would make changes to all those within the p-tag. But lets say I want to add changes to the top and bottom p-tag without affecting any of the other p-tags.This is where classes and IDs come in.

Lets take a look at 'Image B'. We have now added a class to the top and bottom tags so now when we make want to make changes to those two p-tags using our css file it will change both those tags.

Lets now take a look at IDs. In 'Image C' you can see that we have our p-tags within a group-tag that has a class called text. Now all changes made in that text class will affect all of these but lets say you want the middle p-tag to have a different color and as you can see in 'Image C' I've added a ID called color. This will overwrite the class and change that specific ID.

At the end of the day it really is up to you but remember it is best practice to only use IDs if you want a style to apply to one element on the web page, and to use classes if you want a style to apply to multiple elements.
