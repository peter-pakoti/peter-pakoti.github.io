A simple problem I was stuck on was in the Javascript Kata exercise where I had to detemine whether a number was devisable by 3 or 5. It wasn't until Joseph uploaded a video he had done explaining what it was asking and then what we needed to do. It was nice having someone to clear that up and give a bit more in depth understanding to what it is they were after, it made me feel more confident in working on the next few katas alone.

Joesphs helpful video taught me a lot, for example, if we take a look at the image on the left we can see that you don't NEED to use else or else if statements which is something I never knew. He also explained clearly how the modulo operator (%) is used and what it does, which gives back the remainder after interger division. One last thing that I learnt is that the order matters in your if else statements. If we refer to the image on the left, if I had writen that first if statement at the bottom it wouldn't work because it would either return 'Fizz' or 'Buzz' and once a function is done, that's it, it moves on. So by putting it at the top it will run that first and if that's not true then will continue till it returns true. (Hopefully that makes sense and you can understand what I'm trying to say...)
